# AI 辅助短视频剪辑系统 — 系统架构文档

> 版本：v0.1 | 日期：2026-04-08 | 技术栈：Vue3 + Python FastAPI

## 1. 系统整体架构概述

本系统采用**五层架构**，AI 作为辅助引擎嵌入业务流程，人工始终保有最终决策权。

```
┌──────────────────────────────────────────────────────────────────┐
│                        前端展示层（Vue3）                          │
│   问答引导界面 │ 剧本编辑器 │ 素材管理 │ 时间轴编辑 │ 导出管理     │
└─────────────────────────────┬────────────────────────────────────┘
                              │ HTTP / WebSocket
┌─────────────────────────────▼────────────────────────────────────┐
│                       API 网关层（FastAPI）                        │
│          路由分发 │ 鉴权（JWT）│ 限流 │ 请求日志                   │
└──────┬──────────────┬──────────────┬──────────────┬──────────────┘
       │              │              │              │
┌──────▼──────┐ ┌─────▼──────┐ ┌────▼──────┐ ┌────▼──────────────┐
│  会话 &      │ │  剧本生成   │ │ 视频获取  │ │   剪辑 & 渲染     │
│  引导服务   │ │   服务      │ │   服务    │ │     服务          │
└──────┬──────┘ └─────┬──────┘ └────┬──────┘ └────┬──────────────┘
       │              │              │              │
┌──────▼──────────────▼──────────────▼──────────────▼──────────────┐
│                        AI 引擎层                                   │
│   LLM 对话引擎 │ 视觉理解模型 │ ASR 语音识别 │ 音频分析引擎         │
└─────────────────────────────┬────────────────────────────────────┘
                              │
┌─────────────────────────────▼────────────────────────────────────┐
│                         数据存储层                                  │
│   PostgreSQL │ 对象存储（MinIO/OSS）│ Redis │ 消息队列（Celery）    │
└──────────────────────────────────────────────────────────────────┘

## 2. 模块划分详解

### 2.1 前端模块（Vue3 + Vite）

#### 🧭 引导式问答界面
- 进度条展示当前访谈阶段（共 N 步）
- 混合题型：单选、多选、文字填空、滑块（情绪强度）
- 支持返回上一步修改答案
- 实时将答案流式传给后端，动态生成下一个问题
- 最终确认页：展示 AI 总结的创作方向，人工一键确认或修改

#### 📝 剧本编辑器
- 支持多种内置格式模板：
  - 分镜脚本（镜号、时长、画面描述、台词、备注）
  - 口播文案（段落 + 配图说明）
  - 标准剧本格式（场景 + 人物 + 动作 + 对话）
- 支持用户自定义模板（字段定义 + 排版配置）
- 富文本编辑，支持 Markdown 快捷键
- 导出为 PDF / DOCX / JSON

#### 🗂️ 素材管理
- 本地文件上传（拖拽，支持 MP4/MOV/AVI，最大 2GB）
- 链接输入（抖音分享链接，调用火山引擎 API 解析下载）
- 素材预览、标签管理、批量操作

#### ✂️ 视频预览与人工编辑界面
- 视频播放器（自定义控件）
- 时间轴组件：显示 AI 自动打点结果（场景片段、字幕、BGM 轨道）
- 人工操作：拖动调整切点、删除/合并片段、修改字幕文本、替换 BGM
- 转场选择面板（淡入淡出、叠化、推拉等）
- 实时预览合成效果

#### 📤 导出管理界面
- 分辨率选择（1080P / 720P / 抖音竖屏 9:16）
- 水印设置
- 导出进度实时展示（WebSocket 推送）
- 历史导出记录

---

### 2.2 后端模块（Python FastAPI）

#### 🔄 会话管理服务（Session & Workflow Service）
```
职责：维护引导流程的状态机
- 创建/恢复引导会话
- 存储用户每步答案
- 根据答案动态决定下一个问题（决策树 or LLM 动态生成）
- 会话完成后输出结构化创作简报（Brief JSON）
```

#### 📖 剧本生成服务（Script Generation Service）
```
职责：将创作简报转换为可编辑剧本
- 接收 Brief JSON，调用 LLM 生成初稿
- 支持按模板格式化输出
- 版本管理：每次 AI 生成和人工修改均保存版本快照
- 多轮修改：人工可针对某段落让 AI 重新生成
```

#### 📥 视频获取服务（Video Ingestion Service）
```
职责：统一处理视频来源
- 本地上传：接收文件流，存储至对象存储，提取元数据
- 抖音链接：
  1. 解析分享链接中的 video_id
  2. 调用火山引擎 veVOD API 获取无水印视频地址
  3. 异步下载并入库
- 格式转码：统一转为 H.264 MP4 供后续处理
```

#### 🤖 AI 剪辑引擎（AI Clip Engine）
```
职责：对视频进行智能分析和自动剪辑
工作流（异步 Celery 任务）：
  1. 视频拆帧（每秒 N 帧）
  2. 场景识别 → 生成场景时间戳列表
  3. ASR 语音识别 → 生成字幕 SRT
  4. 情绪/节奏分析 → 推荐切点
  5. BGM 匹配 → 从曲库中选取匹配情绪的背景音乐
  6. 生成剪辑方案 JSON（片段列表 + 转场 + 字幕 + BGM）
```

#### 👤 人工审核与覆写服务（Review & Override Service）
```
职责：接收前端人工修改，更新剪辑方案
- 保存每次人工修改的 diff
- 支持撤销/重做
- 锁定机制（防止并发修改）
```

#### 🎬 导出渲染服务（Export & Render Service）
```
职责：将剪辑方案渲染为成品视频
- 读取最终剪辑方案 JSON
- 使用 FFmpeg 执行：片段拼接、字幕烧录、BGM 混音、转场合成
- 支持多分辨率导出
- 导出完成后通过 WebSocket 通知前端，提供下载链接
```

### 2.3 AI 引擎层

| 引擎 | 能力 | 推荐方案 |
|------|------|----------|
| LLM 对话引擎 | 深度访谈问答、剧本生成、段落重写 | OpenAI GPT-4o / 混元大模型 |
| 视觉理解模型 | 场景分割、内容识别、情绪识别 | 自部署 PySceneDetect + CLIP |
| ASR 语音识别 | 字幕生成 | Whisper (OpenAI) / 腾讯云 ASR |
| 音频分析 | 节奏检测、BGM 情绪匹配 | Librosa + Essentia |

### 2.4 数据存储层

| 组件 | 用途 | 推荐方案 |
|------|------|----------|
| 关系型数据库 | 用户、项目、剧本、剪辑任务、版本 | PostgreSQL |
| 对象存储 | 原始视频、成品、素材、字幕文件 | MinIO（自建）/ 阿里云 OSS |

| 消息队列 | 异步剪辑任务调度 | Celery + Redis Broker |

## 3. 核心数据流

### 流程一：AI 引导创作流程

```
用户打开新项目
      │
      ▼
引导服务创建会话，推送第一个问题
      │
      ▼
用户回答（选择/填写）→ 前端发送答案
      │
      ▼
引导服务更新状态机，LLM 动态生成下一问题（共约 8-12 步）
      │
      ▼
所有问题回答完毕 → 引导服务生成 Brief JSON
      │
      ▼
剧本生成服务接收 Brief → 调用 LLM 生成剧本初稿
      │
      ▼
前端展示剧本 → 人工审阅/修改/确认
      │
      ▼
剧本版本锁定，进入素材准备阶段
```

### 流程二：视频剪辑流程

```
用户上传视频 / 输入抖音链接
      │
      ▼
视频获取服务处理 → 存入对象存储，生成 video_id
      │
      ▼
用户提交剪辑请求（关联剧本 + 视频素材）
      │
      ▼
Celery 任务入队 → AI 剪辑引擎异步执行
  ├─ 场景识别 & 打点
  ├─ ASR 字幕生成
  ├─ BGM 推荐
  └─ 生成剪辑方案 JSON
      │
      ▼
WebSocket 推送进度 → 完成后前端加载时间轴
      │
      ▼
人工在时间轴上调整（移动切点、修改字幕、换 BGM、加转场）
      │
      ▼
确认终稿 → 提交导出任务
      │
      ▼
渲染服务执行 FFmpeg 合成 → 生成成品 MP4
      │
      ▼
WebSocket 通知完成 → 用户下载
```

## 4. 火山引擎 API 集成说明

### 使用场景
抖音视频通过分享链接（`https://v.douyin.com/xxx`）输入时，需要：
1. 解析出真实视频 ID
2. 获取无水印视频下载地址

    # 1. 跟随重定向获取真实 URL
    # 2. 提取 video_id 参数
    ...

def download_douyin_video(video_id: str) -> str:
    """调用火山引擎 veVOD API 获取播放地址"""
    vod_service = VodService()
    vod_service.set_ak(VOLC_ACCESS_KEY)
    vod_service.set_sk(VOLC_SECRET_KEY)
    
    resp = vod_service.GetPlayInfo({
        "Vid": video_id,
        "Ssl": 1,
    })
    return resp["Result"]["PlayInfoList"][0]["MainPlayUrl"]
```

### 相关 API
- **视频点播（veVOD）**：获取播放地址、视频元数据
- **内容审核（veImageX/内容安全）**：视频内容合规检测（可选）
- **视频理解（veVideoX）**：视频标签、场景识别（可选，作为 AI 引擎补充）

## 5. 关键技术选型

| 模块 | 技术/库 | 说明 |
|------|---------|------|
| 前端框架 | Vue3 + Vite + TypeScript | 主框架 |
| 前端状态管理 | Pinia | 问答会话状态、时间轴状态 |
| 时间轴组件 | 自研 or wavesurfer.js 改造 | 视频剪辑时间轴 |
| 富文本编辑 | Tiptap | 剧本编辑器 |
| 后端框架 | FastAPI + Uvicorn | 异步高性能 API |
| 异步任务 | Celery + Redis | AI 剪辑任务队列 |
| 视频处理 | FFmpeg（via ffmpeg-python） | 剪辑渲染合成 |
| 场景识别 | PySceneDetect | 自动场景切割 |
| 语音识别 | OpenAI Whisper | 字幕生成 |
| 音频分析 | Librosa | BGM 节拍/情绪分析 |
| LLM 调用 | LangChain + OpenAI SDK | 问答引擎 & 剧本生成 |
| 数据库 ORM | SQLAlchemy + Alembic | DB 操作 & 迁移 |
| 实时通信 | FastAPI WebSocket | 任务进度推送 |
| 对象存储 | MinIO Python SDK | 视频文件存储 |
| 容器化 | Docker + Docker Compose | 本地 & 生产部署 |
